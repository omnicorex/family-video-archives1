# Family Archives — Instructions

A two-part system: Python scripts that pull video data from your MediaCMS server,
and a static website that displays the result as curated playlists.

---

## Project Layout

```
family-video-archives1/
├── scripts/
│   ├── 01_fetch_videos.py     ← fetches from MediaCMS → data/videos.csv
│   └── 02_csv_to_json.py      ← converts edited CSV  → site/data/playlists.json
├── data/
│   └── videos.csv             ← you edit this manually in Excel / Google Sheets
└── site/                      ← the website (open index.html in a browser)
    ├── index.html
    ├── playlist.html
    ├── css/styles.css
    ├── js/main.js
    ├── data/
    │   └── playlists.json     ← auto-generated by script 02
    └── branding/
        ├── config.json        ← site name, tagline, colors — edit freely
        ├── logo.svg           ← swap for your own SVG logo
        ├── favicon.svg        ← swap for your own favicon
        ├── og-image.svg       ← social-card image (export to og-image.png)
        └── site.webmanifest
```

## Step-by-Step Workflow

### 1. First-time setup (once)

Make sure Python 3.7+ is installed. No extra packages needed — the scripts
use only the standard library.

```
python --version
```

### 2. Fetch videos from MediaCMS

```
python scripts/01_fetch_videos.py
```

This talks to `https://tube.tbg2.cloud/api/v1/media/?author=Family` (with
pagination) and saves every video's metadata to **`data/videos.csv`**.

Output columns:

| Column | What it is |
|--------|-----------|
| `id` | MediaCMS `friendly_token` |
| `title` | Video title from MediaCMS |
| `url` | Watch page URL |
| `embed_url` | Embed iframe URL |
| `thumbnail` | Thumbnail image URL |
| `duration_seconds` | Duration in seconds |
| `date_added` | Date uploaded |
| `original_description` | Description from MediaCMS (may be empty) |
| `playlist` | **YOU fill this in** |
| `order` | **YOU fill this in** |
| `custom_description` | **YOU fill this in** |

### 3. Edit the CSV

Open **`data/videos.csv`** in Excel, Google Sheets, or LibreOffice Calc.

For each video, fill in the three rightmost columns:

- **`playlist`** — the group/chapter name, e.g. `Christmas 1990s`
  All videos sharing the same name become one playlist.
  Leave blank to put the video in an "Ungrouped" catch-all playlist.

- **`order`** — a number controlling sort order within the playlist (1, 2, 3 …).
  Videos with the same order number sort alphabetically.

- **`custom_description`** — your own description text shown on the site.
  If left blank, the `original_description` from MediaCMS is used instead.

**Save as CSV** when done (not XLSX).

### 4. Generate the website JSON

```
python scripts/02_csv_to_json.py
```

Reads the CSV and writes **`site/data/playlists.json`**.
The script prints a summary of all playlists and video counts.

### 5. View the website

The site is a plain static site — it needs a simple local web server
(not just double-clicking the HTML file, because browser security blocks
local `fetch()` calls from `file://` URLs).

**Easiest option — Python built-in server:**

```
cd site
python -m http.server 8080
```

Then open: http://localhost:8080

**VS Code option:**
Install the "Live Server" extension, right-click `site/index.html` → Open with Live Server.

**For permanent hosting:**
Upload the entire `site/` folder to any static host:
Netlify, Vercel, GitHub Pages, Cloudflare Pages, or your own web server.

## Updating Videos Later

Whenever you add or change videos on MediaCMS, just repeat steps 2–4:

```
python scripts/01_fetch_videos.py   # re-fetch (adds new videos, keeps existing rows)
# Edit data/videos.csv again if needed
python scripts/02_csv_to_json.py    # regenerate JSON
```

> **Note:** Script 01 re-fetches everything fresh. Your edits to `playlist`,
> `order`, and `custom_description` will be lost unless you re-apply them.
> **Tip:** Keep a backup copy of your edited CSV, or use a spreadsheet with a
> separate "master" sheet where you maintain your custom data and use VLOOKUP /
> INDEX-MATCH to merge it with the newly fetched data.

## Customising the Site

### Site name, tagline, colors

Edit **`site/branding/config.json`** — changes take effect on the next page load,
no rebuild needed.

```jsonc
{
  "siteName":   "Our Family Memories",   // shown in nav, hero, title bar
  "tagline":    "Four generations …",    // hero subtitle
  "colors": {
    "red":    "#C8001A",   // Armenian red  — primary accent
    "blue":   "#002FA0",   // Armenian blue — secondary
    "orange": "#D88000"    // Armenian orange — tertiary
  }
}
```

### Playlist descriptions

In `config.json`, add a `playlist_descriptions` object:

```jsonc
"playlist_descriptions": {
  "Christmas 1990s": "Home videos from Christmas gatherings in the nineties.",
  "Summer Holidays": "Trips to the beach, mountains, and everything in between."
}
```

The description appears on the home page card and at the top of the playlist page.

### Logo

Replace **`site/branding/logo.svg`** with your own SVG.
Recommended size: 40×40 viewBox, works at 34px display size.

### Favicon

Replace **`site/branding/favicon.svg`** with your own.
For best browser support, also export a 32×32 PNG as **`favicon.png`**
and a 180×180 PNG as **`apple-touch-icon.png`** in the same folder.

### Social card (OG image)

The file **`site/branding/og-image.svg`** is the source artwork.
Export it as a **1200×630 PNG** named **`og-image.png`** in the same folder
(browsers and social platforms need a raster PNG, not SVG, for OG images).

You can export it with:
- Inkscape: File → Export PNG Image → set 1200×630
- Any browser: open the SVG, right-click → Save As, then convert
- Online tools: svgtopng.com or similar

## Troubleshooting

**Script 01 returns 0 videos:**
- Check the `USERNAME` constant at the top of the script matches the exact
  MediaCMS username (case-sensitive).
- Confirm the user has public (not private) videos.
- Try opening `https://tube.tbg2.cloud/api/v1/media/?author=Family` in your
  browser to see the raw API response.

**Videos show "embed not available":**
- MediaCMS must be configured to allow embedding. Check the site admin settings.

**Website shows "No playlists yet":**
- Make sure you've run script 02 and that `site/data/playlists.json` is not empty.
- Check you're serving from a web server (not opening the file directly).

**Colors look wrong:**
- Edit the hex values in `site/branding/config.json` and reload.
