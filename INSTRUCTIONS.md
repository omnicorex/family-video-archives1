# Family Archives — Instructions

A two-part system: Python scripts that sync video data between your MediaCMS
server and a Grist spreadsheet, and a static website that displays the result
as curated playlists.

---

## Project Layout

```
family-video-archives1/
├── scripts/
│   ├── 03_grist_sync.py       ← syncs MediaCMS metadata → Grist
│   ├── 04_extract_dates.py    ← suggests dates from video titles → Grist
│   └── 05_grist_to_json.py    ← exports curated Grist data → site JSON
├── data/
│   └── backups/               ← auto-created; timestamped backups before each write
└── site/                      ← the website (serve with a local web server)
    ├── index.html
    ├── playlist.html
    ├── css/styles.css
    ├── js/main.js
    ├── data/
    │   └── playlists.json     ← auto-generated by script 05
    └── branding/
        ├── config.json        ← site name, tagline, colors — edit freely
        ├── logo.svg           ← swap for your own SVG logo
        ├── favicon.svg        ← swap for your own favicon
        ├── og-image.svg       ← social-card image (export to og-image.png)
        └── site.webmanifest
```

## Grist Spreadsheet

Your video data lives in Grist at:

- **URL:** https://sheets.tbg2.cloud
- **Document:** Family_Video_Archives
- **Table:** Table1

The table has these columns:

| Column | Managed by | What it is |
|--------|-----------|------------|
| `media_id` | Script 03 | MediaCMS ID — master key, never changes |
| `title` | Script 03 | Video title from MediaCMS |
| `description` | Script 03 | Description from MediaCMS |
| `thumbnail_url` | Script 03 | Thumbnail image URL |
| `hls_url` | Script 03 | HLS stream URL for video playback |
| `suggested_date` | Script 04 | Date extracted from title — review and copy to `date` if correct |
| `date` | **You** | Event date shown on the site (yyyy-mm-dd or yyyy-mm or yyyy) |
| `playlist` | **You** | Playlist/group name — all videos with the same name form one playlist |
| `order` | **You** | Sort order within the playlist (1, 2, 3 …) |
| `custom_title` | **You** | Override the MediaCMS title — leave blank to use original |
| `custom_description` | **You** | Override the MediaCMS description — leave blank to use original |

> Script 03 only ever writes to its own five columns. Your custom columns
> (`date`, `playlist`, `order`, `custom_title`, `custom_description`) are
> **never overwritten** no matter how many times you run it.

## Step-by-Step Workflow

### 1. First-time setup (once)

Make sure Python 3.7+ is installed. No extra packages needed — all scripts
use only the standard library.

```
py --version
```

### 2. Sync videos from MediaCMS → Grist

```
py scripts/03_grist_sync.py
```

You will be prompted for your MediaCMS password. The script then:

1. Creates backups of the current Grist table (`data/backups/grist_*.csv`)
   and the current `site/data/playlists.json` before touching anything.
2. Fetches all videos from MediaCMS (including unlisted ones).
3. Compares with what is already in Grist:
   - **New videos** are added as new rows.
   - **Changed videos** have only their MediaCMS fields updated.
   - **Unchanged videos** are skipped entirely.
4. Your custom columns are never touched.

### 3. Fill in your custom data in Grist

Open https://sheets.tbg2.cloud and edit the table directly.

For each video, fill in:

- **`playlist`** — the group name, e.g. `Christmas 1990s`.
  All videos sharing the same name become one playlist on the site.
  Leave blank to put a video in an "Ungrouped" catch-all playlist.

- **`order`** — a number controlling sort order within the playlist (1, 2, 3 …).
  Videos with no order set fall to the end, sorted by title.

- **`date`** — the date the event happened, e.g. `1985-07-04`.
  Partial dates like `1985-07` or just `1985` are also fine.
  This is shown on the site next to each video.

- **`custom_title`** — leave blank to use the MediaCMS title as-is.

- **`custom_description`** — leave blank to use the MediaCMS description as-is.

### 4. (Optional) Auto-suggest dates from titles

```
py scripts/04_extract_dates.py
```

This scans each video title and tries to detect a date. Results are written
to the `suggested_date` column — it prints a preview table so you can see
what was found before committing to anything.

**The `date` column is never touched.** Review `suggested_date` in Grist
and copy any dates you want into `date` by hand.

Recognised formats include:
- `1985-07-04`, `July 4, 1985`, `Jul 4th 1985`, `7/4/1985`
- `July 1985`, `Summer 1985` (approximate)
- `1985` (year only)

### 5. Generate the website JSON

```
py scripts/05_grist_to_json.py
```

Reads your curated Grist table and writes **`site/data/playlists.json`**.
A backup of the previous JSON is saved to `data/backups/` first.

Field priority when building the JSON:
- Title → `custom_title` if set, otherwise `title`
- Description → `custom_description` if set, otherwise `description`

### 6. View the website

The site is a plain static site — it needs a local web server (not just
double-clicking the HTML file, because browsers block `fetch()` calls from
`file://` URLs).

**Easiest option — Python built-in server:**

```
cd site
py -m http.server 8080
```

Then open: http://localhost:8080

**VS Code option:**
Install the "Live Server" extension, right-click `site/index.html` → Open with Live Server.

**For permanent hosting:**
Upload the entire `site/` folder to any static host:
Netlify, Vercel, GitHub Pages, Cloudflare Pages, or your own web server.

## Updating Videos Later

Whenever you add or change videos on MediaCMS, repeat steps 2, 5 (and
optionally 4):

```
py scripts/03_grist_sync.py      # pulls in new/changed videos; never loses your edits
py scripts/04_extract_dates.py   # optional — re-run to get date suggestions for new videos
py scripts/05_grist_to_json.py   # rebuilds the site JSON
```

Your custom data in Grist (`date`, `playlist`, `order`, `custom_title`,
`custom_description`) is always preserved — script 03 only updates what
changed on the MediaCMS side.

## Customising the Site

### Site name, tagline, colors

Edit **`site/branding/config.json`** — changes take effect on the next page
load, no rebuild needed.

```jsonc
{
  "siteName":   "Our Family Memories",   // shown in nav, hero, title bar
  "tagline":    "Four generations …",    // hero subtitle
  "colors": {
    "red":    "#C8001A",   // primary accent
    "blue":   "#002FA0",   // secondary
    "orange": "#D88000"    // tertiary
  }
}
```

### Playlist descriptions

In `config.json`, add a `playlist_descriptions` object. These descriptions
override whatever is in the JSON and appear on the home page card and at the
top of the playlist page.

```jsonc
"playlist_descriptions": {
  "Christmas 1990s": "Home videos from Christmas gatherings in the nineties.",
  "Summer Holidays": "Trips to the beach, mountains, and everything in between."
}
```

### Logo

Replace **`site/branding/logo.svg`** with your own SVG.
Recommended: 40×40 viewBox, works at 34px display size.

### Favicon

Replace **`site/branding/favicon.svg`** with your own.
For best browser support, also export a 32×32 PNG as **`favicon.png`**
and a 180×180 PNG as **`apple-touch-icon.png`** in the same folder.

### Social card (OG image)

The file **`site/branding/og-image.svg`** is the source artwork.
Export it as a **1200×630 PNG** named **`og-image.png`** in the same folder
(social platforms require a raster PNG, not SVG).

## Troubleshooting

**Script 03 returns 0 videos:**
- Check the `MEDIACMS_USER` constant at the top of the script matches the
  exact MediaCMS username (case-sensitive).
- Make sure the password you enter is correct.
- Try opening `https://tube.tbg2.cloud/api/v1/media/?author=family` in your
  browser to confirm the API is reachable.

**Videos won't play:**
- Make sure you have run script 03 (which populates `hls_url`) and then
  script 05 (which puts `hls_url` into the site JSON).
- Check `site/data/playlists.json` — each video should have a non-empty
  `hls_url` field.
- Do a hard refresh in the browser (`Ctrl+Shift+R`) to clear cached JS/JSON.

**Website shows "No playlists yet":**
- Make sure you have run script 05 and that `site/data/playlists.json` exists
  and is not empty.
- Confirm you are serving from a web server, not opening the HTML file directly.
- Check that at least one video in Grist has a `playlist` value filled in.

**Colors look wrong:**
- Edit the hex values in `site/branding/config.json` and reload.

**Grist API errors:**
- Confirm the `GRIST_API_KEY` in the scripts matches your Grist account key.
- Check that `GRIST_TABLE_ID` is still `Table1` — if you renamed the table
  in the Grist UI you will need to update this constant in all three scripts.
